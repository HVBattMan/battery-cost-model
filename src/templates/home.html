Battery Cost App by Ethan Woodbury

An application for modeling the cost of lithium-ion batteries


How to Use


Input

1. Cell Input

Customize a cell design to your desired specifications by setting metrics like electrode area, N-P ratio, and active material capacity.

Input

2. Price Input

Add customized prices for all cell components: active materials, binders, current collectors, etc. Alternatively, use a default price dataset.

Calculate

3. Run Model

Choose a cell design and price dataset to plug into the model to calculate the total battery cost. View model as pie chart.

Visualize

4. Compare Models

Plot side-by-side cost stacks to compare total cell-level and pack-level costs of previously-created battery models.

If you don't want to make your own models, you can go straight to viewing previously-created models in Compare Models.



About


Model Methodology

The model first uses the input cell parameters to calculate the energy content of each cell and the material required per cell. This is then used to calculate the material required on a per kWh basis. Finally, the input price dataset is used to calculate the total cost per kilowatt-hour. Model built in Python.
